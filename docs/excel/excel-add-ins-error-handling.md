# <a name="error-handling-with-the-excel-javascript-api"></a>Обработка ошибок с помощью Excel JavaScript

При создании надстройки с использованием API JavaScript для Excel не забудьте включить логику для обработки ошибок, возникающих в среде выполнения. Это очень важно из-за асинхронного характера API.

> [!NOTE]
> `sync()` Дополнительные сведения о методе и асинхронном характере Excel JavaScript см. в Excel объектной модели [JavaScript](excel-add-ins-core-concepts.md) в Office надстройки.

## <a name="best-practices"></a>Рекомендации

В [примерах](https://github.com/OfficeDev/Office-Add-in-samples) кода [и Script Lab](../overview/explore-with-script-lab.md) фрагментах кода вы заметите, `Excel.run` `catch` `Excel.run`что каждый вызов сопровождается оператором для перехвата всех ошибок, которые возникают внутри . Мы рекомендуем использовать этот шаблон, когда вы будете создавать надстройки с использованием API JavaScript для Excel.

```js
$("#run").click(() => tryCatch(run));

async function run() {
  await Excel.run(async (context) => {
      // Add your Excel JavaScript API calls here.

      // Await the completion of context.sync() before continuing.
    await context.sync();
    console.log("Finished!");
  });
}

/** Default helper for invoking an action and handling errors. */
async function tryCatch(callback) {
  try {
    await callback();
  } catch (error) {
    // Note: In a production add-in, you'd want to notify the user through your add-in's UI.
    console.error(error);
  }
}

```

## <a name="api-errors"></a>Ошибки API

Если Excel API JavaScript не выполняется успешно, API возвращает объект ошибки, содержащий следующие свойства.

- **code**.  Свойство `code` сообщения об ошибке содержит строку, входящую в список `OfficeExtension.ErrorCodes` или `Excel.ErrorCodes`. Например, код ошибки InvalidReference указывает, что ссылка недопустима для указанной операции. Коды ошибок не локализованы.

- **message.** Свойство `message` сообщения об ошибке содержит сводные сведения об ошибке в локализованной строке. Сообщение об ошибке не предназначено для пользователей. Код ошибки и соответствующую бизнес-логику следует использовать для определения сообщения об ошибке, которое ваша надстройка будет отображать для пользователей.

- **debugInfo.** Если в сообщении об ошибке имеется свойство `debugInfo`, в нем содержатся дополнительные сведения, которые вы можете использовать, чтобы понять причину ошибки.

> [!NOTE]
> Если вы используете `console.log()` печать сообщений об ошибках в консоли, эти сообщения отображаются только на сервере. Конечные пользователи не видят эти сообщения об ошибках в области задач надстройки или в любом месте Office приложения. Сведения об ошибках для пользователя см. в [разделе "Уведомления об ошибках"](#error-notifications).

## <a name="error-notifications"></a>Уведомления об ошибках

Способ создания отчетов об ошибках для пользователей зависит от системы пользовательского интерфейса, которую вы используете. Если вы используете React в качестве системы пользовательского интерфейса, используйте компоненты Fluent пользовательского интерфейса и элементы проектирования. Выберите соответствующий элемент управления на этой Fluent [пользовательского интерфейса](https://developer.microsoft.com/fluentui#/controls/web). Мы рекомендуем передавать сообщения об ошибках с помощью панели сообщений, диалогового окна или модального сообщения. Если ошибка находится во входных данных пользователя, выведите ее полужирным красным цветом рядом с элементом управления вводом. Пример [Office-Add-in-Microsoft-Graph-React](https://github.com/OfficeDev/PnP-OfficeAddins/tree/0706cc67645675a48747f1fec1b1e5722b575b11/Samples/auth/Office-Add-in-Microsoft-Graph-React) использует элемент MessageBar и изменяет его для учета меню личных данных в области задач надстройки.

Если вы не используете React пользовательского интерфейса, рассмотрите возможность использования старых компонентов пользовательского интерфейса Fabric, реализованных непосредственно в HTML и JavaScript. Некоторые примеры шаблонов находятся в [репозитории Office-Add-In-UX-Design-Patterns-Code](https://github.com/OfficeDev/Office-Add-in-UX-Design-Patterns-Code/tree/master/templates). Обратите особое значение на вложенные папки диалогового окна и навигации. В [примере Excel-Add-in-SalesLeads](https://github.com/OfficeDev/Excel-Add-in-SalesLeads) используется баннер сообщения.
