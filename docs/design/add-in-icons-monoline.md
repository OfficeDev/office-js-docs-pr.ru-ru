# <a name="monoline-style-icon-guidelines-for-office-add-ins"></a>Руководство по значкам моностройного стиля для надстройки Office

В приложениях Office используются монолинейные значки стилей. Если вы предпочитаете, чтобы ваши значки совпадали со стилем "Новый" office 2013 и office 2013 и более новыми, см. рекомендации по значкам "Новый стиль" для надстройки [Office.](add-in-icons-fresh.md)

## <a name="office-monoline-visual-style"></a>Визуальный стиль монолайна Office

Цель стиля Monoline — обеспечить единообразие, четкое и доступное оформление для взаимодействия действий и функций с помощью простых визуальных объектов, обеспечить доступность значков для всех пользователей и иметь стиль, соответствующий стилю, который используется в других частях Windows.

Ниже представлены рекомендации для сторонних разработчиков, которые хотят создавать значки для функций, которые будут соответствовать значкам, уже представленным в продуктах Office.

### <a name="design-principles"></a>Принципы разработки

- Простое, чистое, понятное.
- Содержит только необходимые элементы.
- Навеяно стилем значка Windows.
- Доступно для всех пользователей.

#### <a name="conveying-meaning"></a>Передав значение

- Используйте описательные элементы, например страницу, для представления документа или конверта для представления почты.
- Используйте один и тот же элемент для представления одной и той же концепции, то есть почта всегда представлена конвертом, а не отметкой.
- Используйте основную метафору во время разработки концепции.

#### <a name="reduction-of-elements"></a>Сокращение элементов

- Уменьшите значение значка до его основного значения, используя только элементы, необходимые для метафоры.
- Ограничив количество элементов значка двумя, независимо от размера значка.

#### <a name="consistency"></a>Согласованность

Размеры, расположение и цвет значков должны быть согласованы.

#### <a name="styling"></a>Стиль

##### <a name="perspective"></a>Perspective

По умолчанию монолинейные значки перенадвигаются вперед. Некоторые элементы, которые требуют перспективы и/или поворота, например куб, разрешены, но исключения должны быть с минимальными.

##### <a name="embellishment"></a>Безвластие

Монолайн — это чистый минимальный стиль. Все использует плоский цвет, то есть градиенты, текстуры и источники света не существуют.

## <a name="designing"></a>Проектирование

### <a name="sizes"></a>Размеры

Мы рекомендуем создавать каждый значок всех этих размеров для поддержки устройств с высоким DPI. Абсолютно *необходимые* размеры: 16, 20 и 32 пкс, так как это 100% размеров.

**16 px, 20 px, 24 px, 32 px, 40 px, 48 px, 64 px, 80 px, 96 px**

#### <a name="elements"></a>Элементы

- **Base**: основная концепция, представленная значком. Обычно это единственный визуальный элемент, необходимый для значка, но иногда основную концепцию можно расширить с помощью дополнительного элемента модификатора.

- **Модификатор** Любой элемент, наложение базового; то есть модификатор, который обычно представляет действие или состояние. Он изменяет базовый элемент, выступая в качестве дополнения, изменения или дескриптора.

#### <a name="cutouts"></a>Cutouts

Когда элемент значка помещается поверх другого элемента, вырезание (нижнего элемента) используется для предоставления пространства между двумя элементами, в основном в целях учитаемости. Обычно это происходит, когда модификатор помещается поверх базового элемента, но также есть случаи, когда ни один из элементов не является модификатором. Эти вырезания между двумя элементами иногда называются разрывом.

Размер разрывов должен быть такой же шириной, как и вес линии, используемый для этого размера. Если значок составляет 16 пкс, ширина разрывов будет 1 пк, а если это значок 48 пкс, разрыв должен быть 2 пк. В следующем примере показан значок 32 пкс с разрывом в 1 пкс между модификатором и базовой базой.

![Значок 32 пкс с разрывом в 1 пкс между модификатором и базовой базой](../images/monolineicon25.png)

В некоторых случаях разрыв может увеличиться на 1/2 пкс, если модификатор имеет диагональный или кривый край, а стандартный разрыв не обеспечивает достаточного разделения. Скорее всего, это повлияет только на значки с весом 1 пкс: 16 пк, 20 пкс, 24 пкс и 32 пкс.

Большинство значков в наборе значков Monoline требуют заполнения фона. Однако в некоторых случаях у объекта не будет заливки, поэтому заливка не должна применяться. Следующие значки имеют белый заливки.

![Компиляция пяти значков с белыми заливками](../images/monolineicon26.png)

Следующие значки не заполняются. (Значок шестеренки включен, чтобы показать, что центральное изображение не заполнено.)

![Компиляция пяти значков без заливки](../images/monolineicon27.png)

##### <a name="best-practices-for-fills"></a>Best practices for fills

###### <a name="dos"></a>Dos:

- Заполните любой элемент, который имеет запредельную границу и, естественно, будет иметь заливки.
- Используйте отдельную фигуру для создания заливки фона.
- Используйте **фоновую заливку** из [цветовой палитры.](#color)
- Поддерживание разделения пикселей между перекрывающимися элементами.
- Заполните несколько объектов.

###### <a name="donts"></a>Не:

- Не заполняйте объекты, которые не были бы заполнены естественным образом; например, paperclip.
- Не заполняйте скобки.
- Не заполняйте цифры или буквы.

### <a name="color"></a>Цвет

Цветовая палитра разработана для простоты и доступности. Он содержит 4 нейтральных цвета и два варианта для синего, зеленого, желтого, красного и сиреневых. Оранжевый цвет намеренно не включен в цветовую палитру монолайновых значков. Каждый цвет предназначен для использования определенными способами, как описано в этом разделе.

#### <a name="how-to-use-color"></a>Использование цвета

В монолайновой цветовой палитре все цвета имеют автономные варианты, варианты Outline и Fill. Как правило, элементы построены с помощью заливки и границы. Цвета применяются в одном из следующих шаблонов:

- Автономный цвет только для объектов без заливки.
- На границе используется цвет Outline, а для заливки используется цвет заливки.
- На границе используется автономный цвет, а для заливки используется цвет заливки фона.

Ниже приводится пример использования цвета.

![Компиляция трех значков с цветом на границе или заливке или и тем, и другим](../images/monolineicon28.png)

Наиболее распространенной ситуацией является использование элементом standalone темно-серого цвета с фоновой заливки.

При использовании цветного заливки он всегда должен быть с соответствующим цветом Outline. Например, синяя заливка должна использоваться только с синим контуром. Однако есть два исключения из этого общего правила:

- Фоновое заполнение можно использовать с любым автономным цветом.
- Светло-серую заливка можно использовать с двумя разными цветами outline: темно-серым или средним серым.

#### <a name="when-to-use-color"></a>Когда использовать цвет

Цвет следует использовать для передачи значения значка, а не для замеления. Он должен **выделить действие** для пользователя. При добавлении модификатора в базовый элемент с цветом базовый элемент обычно превращается в темно-серый и фоновый заливка, чтобы модификатором был элемент цвета, например в приведенном ниже примере с модификатором "X", добавляемого в базу рисунков в левом значке следующего набора.

![Компиляция пяти значков, которые используют цвет](../images/monolineicon29.png)

Значки следует ограничить одним **дополнительным** цветом, кроме упомянутых выше окрашиваний Outline и Fill. Тем не менее, можно использовать больше цветов, если это важно для его метафоры, с ограничением двух дополнительных цветов, кроме серого. В редких случаях существуют исключения, когда требуется больше цветов. Ниже приводится хороший пример значков, которые используют только один цвет.

  ![Компиляция пяти значков, каждый из которых использует один цвет](../images/monolineicon30.png)

Но в следующих значках используется слишком много цветов.

  ![Компиляция пяти значков, каждый из которых использует несколько цветов](../images/monolineicon31.png)

Используйте **средний серый** цвет для внутреннего "содержимого", например линий сетки в значке таблицы. Дополнительные внутренние цвета используются, когда содержимое должно показывать поведение этого средства управления.

![Компиляция пяти значков со средним серым цветом внутренних элементов](../images/monolineicon32.png)

#### <a name="text-lines"></a>Текстовые строки

Если текстовые строки находятся в "контейнере" (например, текст в документе), используйте средний серый цвет. Текстовые строки, не в контейнере, должны быть **темно-серыми.**

### <a name="text"></a>Текст

Избегайте использования текстовых символов в значках. Так как продукты Office используются по всему миру, мы хотим, чтобы значки были максимально нейтральными на языке.

## <a name="production"></a>Производство

### <a name="icon-file-format"></a>Формат файла значка

Конечные значки должны быть сохранены в PNG-файлах изображений. Используйте формат PNG с прозрачным фоном и 32-битной глубиной.
