# <a name="monoline-style-icon-guidelines-for-office-add-ins"></a>Руководство по значкам стилей Monoline для надстройок Office

Иконография стилей Monoline используется в приложениях Office. Если вы предпочитаете, чтобы значки совпадали со стилем Fresh для Office 2013+, см. рекомендации по значкам в стиле Fresh для [надстройок Office.](add-in-icons-fresh.md)

## <a name="office-monoline-visual-style"></a>Визуальный стиль Office Monoline

Цель стиля Monoline — иметь согласованную, четкую и доступную иконографию для связи действий и функций с помощью простых визуальных эффектов, обеспечения доступности значков для всех пользователей и стиля, соответствующего тем, которые используются в других частях Windows.

Ниже представлены рекомендации для сторонних разработчиков, которые хотят создать значки для функций, которые будут соответствовать значкам, уже представленным продуктам Office.

### <a name="design-principles"></a>Принципы разработки

- Простой, чистый, понятный.
- Содержит только необходимые элементы.
- Вдохновленный стилем значка Windows.
- Доступно для всех пользователей.

#### <a name="conveying-meaning"></a>Передающее значение

- Используйте описательные элементы, такие как страница, чтобы представлять документ или конверт для представления почты.
- Используйте один и тот же элемент для представления одной и той же концепции, то есть почта всегда представлена конвертом, а не штампом.
- Используйте основную метафору во время разработки концепции.

#### <a name="reduction-of-elements"></a>Уменьшение элементов

- Уменьшите значок до основного значения, используя только элементы, необходимые для метафоры.
- Ограничить число элементов в значке двумя, независимо от размера значка.

#### <a name="consistency"></a>Согласованность

Размеры, расположение и цвет значков должны быть последовательными.

#### <a name="styling"></a>Стиль

##### <a name="perspective"></a>Perspective

Значки Monoline по умолчанию имеют передовую линию. Допускаются некоторые элементы, которые требуют перспективы и/или вращения, например куба, но исключения следует совмещение.

##### <a name="embellishment"></a>Приукрашивание

Monoline — это чистый минимальный стиль. Все использует плоский цвет, что означает отсутствие градиентов, текстур или источников света.

## <a name="designing"></a>Проектирование

### <a name="sizes"></a>Размеры

Мы рекомендуем создавать каждый значок всех этих размеров для поддержки устройств с высоким уровнем DPI. Абсолютно *необходимые* размеры 16 px, 20 px и 32 px, так как это 100% размеров.

**16 px, 20 px, 24 px, 32 px, 40 px, 48 px, 64 px, 80 px, 96 px**

#### <a name="elements"></a>Элементы

- **База.** Основная концепция, которую представляет значок. Обычно это единственный визуальный элемент, необходимый для значка, но иногда основная концепция может быть улучшена с помощью дополнительного элемента, модификатора.

- **Модификатор** Любой элемент, наложение на базу; то есть модификатор, который обычно представляет действие или состояние. Он изменяет базовый элемент, выступая в качестве добавления, изменения или дескриптора.

#### <a name="cutouts"></a>Вырезы

Когда элемент значка помещается поверх другого элемента, вырез (нижнего элемента) используется для предоставления пространства между двумя элементами, главным образом для целей читаемости. Это обычно происходит, когда модификатор помещается поверх базового элемента, но есть также случаи, когда ни один из элементов не является модификатором. Эти вырезы между двумя элементами иногда называются "разрывом".

Размер зазора должен быть такой же шириной, как и вес строки, используемый для этого размера. Если сделать значок 16 px, ширина пробела будет 1px, а если это значок 48 px, то разрыв должен быть 2px. В следующем примере показан значок 32 px с разрывом в 1px между модификатором и базовой базой.

![Значок 32 px с разрывом 1px между модификатором и базовой базой](../images/monolineicon25.png)

В некоторых случаях разрыв может быть увеличен на 1/2 px, если модификатор имеет диагональный или изогнутый край, а стандартный разрыв не обеспечивает достаточного разделения. Это, скорее всего, повлияет только на значки с весом строки 1px: 16 px, 20 px, 24 px и 32 px.

#### <a name="background-fills"></a>Заполнение фона

Большинство значков в наборе значков Monoline требуют заполнения фона. Однако существуют случаи, когда объект не имеет естественного заполнения, поэтому не следует применять заливки. Следующие значки имеют белый заливки.

![Компиляция пяти значков с белым заливом](../images/monolineicon26.png)

Следующие значки не заполняются. (Значок передач включен, чтобы показать, что центральное отверстие не заполнено.)

![Компиляция пяти значков без заполнения](../images/monolineicon27.png)

##### <a name="best-practices-for-fills"></a>Лучшие практики для заполнения

###### <a name="dos"></a>Dos:

- Заполните любой элемент с определенной границей и естественным образом заполните его.
- Для создания фонового заполнения используйте отдельную форму.
- Используйте **фоновое заполнение** [из цветовой палитры](#color).
- Сохранение разделения пикселей между перекрывающимися элементами.
- Заполните между несколькими объектами.

###### <a name="donts"></a>Не нужно:

- Не заполняйте объекты, которые естественно не заполняются; например, сальто.
- Не заполняйте скобки.
- Не заполняйте номера или альфа-символы.

### <a name="color"></a>Цвет

Цветовая палитра разработана для простоты и доступности. Он содержит 4 нейтральных цвета и два варианта для синего, зеленого, желтого, красного и фиолетового цветов. Оранжевый цвет намеренно не входит в цветовую палитру значков Monoline. Каждый цвет предназначен для использования определенными способами, как описано в этом разделе.

#### <a name="palette"></a>Палитра

![Цветовая палитра в монолине включает в себя оттенок синего, зеленого, желтого, красного и фиолетового для автономных, контурных и заполняемых](../images/monoline-colors.png)

#### <a name="how-to-use-color"></a>Использование цвета

В цветовой палитре Monoline все цвета имеют автономные, контурные и заполняемые варианты. Как правило, элементы построены с заполнием и границей. Цвета применяются в одном из следующих шаблонов:

- Автономный цвет для объектов без заполнения.
- На границе используется цвет Outline, а для заполнения используется цвет Fill.
- На границе используется автономный цвет, а для заполнения используется цвет Фоновое заполнение.

Ниже приводится пример использования цвета.

![Компиляция трех значков с цветом на границе или заполнения или обоих](../images/monolineicon28.png)

Наиболее распространенной ситуацией будет использование элемента Темно-серый автономный с фоновой заливки.

При использовании цветной заливки он всегда должен быть с соответствующим цветом Outline. Например, Blue Fill следует использовать только с помощью Blue Outline. Но есть два исключения из этого общего правила:

- Фоновое заполнение можно использовать с любым автономным цветом.
- Светло-серый заливка может использоваться с двумя разными цветами Outline: темно-серый или средний серый.

#### <a name="when-to-use-color"></a>Когда использовать цвет

Цвет должен использоваться для передачи значения значка, а не для украшения. Он должен **выделить действие** пользователю. При добавлении модификатора в базовый элемент с цветом базовый элемент обычно превращается в темно-серый и фоновый, так что модификатор может быть элементом цвета, например в случае ниже с модификатором "X", который добавляется в базу изображений в левом значке следующего набора.

![Компиляция пяти значков с использованием цвета](../images/monolineicon29.png)

Вы должны ограничить значки **одним дополнительным** цветом, кроме описанных выше набросков и заливки. Тем не менее, больше цветов можно использовать, если это жизненно важно для его метафоры, с ограничением двух дополнительных цветов, кроме серого. В редких случаях существуют исключения, когда требуется больше цветов. Ниже приводится хороший пример значков, которые используют только один цвет.

  ![Компиляция пяти значков, каждый из которых использует один цвет](../images/monolineicon30.png)

Но в следующих значках используется слишком много цветов.

  ![Компиляция пяти значков, каждый из которых использует несколько цветов](../images/monolineicon31.png)

Используйте **medium Gray** для внутреннего "контента", например линий сетки в значке таблицы. Дополнительные цвета интерьера используются, когда содержимое должно показывать поведение управления.

![Компиляция пяти значков со средними серыми элементами интерьера](../images/monolineicon32.png)

#### <a name="text-lines"></a>Текстовые строки

Если текстовые строки находятся в "контейнере" (например, текст на документе), используйте средне-серый цвет. Текстовые строки, не в контейнере, должны быть **темно-серыми.**

### <a name="text"></a>Текст

Избегайте использования текстовых символов в значках. Так как продукты Office используются по всему миру, мы хотим сохранить значки как можно более нейтральными на языке.

## <a name="production"></a>Производство

### <a name="icon-file-format"></a>Формат файла icon

Конечные значки должны быть сохранены в качестве файлов изображений png. Используйте формат PNG с прозрачным фоном и 32-битной глубиной.
