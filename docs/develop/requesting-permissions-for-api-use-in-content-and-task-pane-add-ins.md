# <a name="requesting-permissions-for-api-use-in-content-and-task-pane-add-ins"></a><span data-ttu-id="cdc25-102">Запрос разрешений на использование API в надстройках содержимого и области задач</span><span class="sxs-lookup"><span data-stu-id="cdc25-102">Requesting permissions for API use in content and task pane add-ins</span></span>

<span data-ttu-id="cdc25-103">В этой статье описываются различные уровни разрешений, которые можно объявить в манифесте надстройки содержимого или надстройки области задач, чтобы указать уровень доступа API JavaScript, необходимый вашей надстройке.</span><span class="sxs-lookup"><span data-stu-id="cdc25-103">This article describes the different permission levels that you can declare in your content or task pane add-in's manifest to specify the level of JavaScript API access your add-in requires for its features.</span></span> 




## <a name="permissions-model"></a><span data-ttu-id="cdc25-104">Модель разрешений</span><span class="sxs-lookup"><span data-stu-id="cdc25-104">Permissions model</span></span>


<span data-ttu-id="cdc25-105">5-уровневая модель разрешений API JavaScript — это основа безопасности и конфиденциальности для пользователей надстроек содержимого и надстроек области задач. На рис. 1 показаны пять уровней разрешений API, которые можно объявить в манифесте надстройки.</span><span class="sxs-lookup"><span data-stu-id="cdc25-105">A five-level JavaScript API access-permissions model provides the basis for privacy and security for users of your content and task pane add-ins. Figure 1 shows the five levels of API permissions you can declare in your add-in's manifest.</span></span>


<span data-ttu-id="cdc25-106">*Рис. 1. Пятиуровневая модель разрешений для надстроек содержимого и надстроек области задач*</span><span class="sxs-lookup"><span data-stu-id="cdc25-106">*Figure 1. The five-level permission model for content and task pane add-ins*</span></span>

![Уровни разрешений для приложений области задач](../images/office15-app-sdk-task-pane-app-permission.png)



<span data-ttu-id="cdc25-p101">Эти разрешения задают подмножество API, которые надстройка содержимого или области задач сможет использовать во время выполнения, когда пользователь вставляет, а затем активирует вашу надстройку (доверяет ей). Чтобы объявить уровень разрешений, необходимый вашей надстройке, укажите одно из текстовых значений разрешения в элементе [Permissions](https://docs.microsoft.com/office/dev/add-ins/reference/manifest/permissions?view=office-js) манифеста надстройки. Следующий пример запрашивает разрешение **WriteDocument**, которое разрешает использовать только методы записи в документ (но не методы чтения).</span><span class="sxs-lookup"><span data-stu-id="cdc25-p101">These permissions specify the subset of the API that the add-in runtime will allow your content or task pane add-in to use when a user inserts, and then activates (trusts) your add-in. To declare the permission level your content or task pane add-in requires, specify one of the permission text values in the [Permissions](https://docs.microsoft.com/office/dev/add-ins/reference/manifest/permissions?view=office-js) element of your add-in's manifest. The following example requests the **WriteDocument** permission, which will allow only methods that can write to (but not read) the document.</span></span>




```XML
<Permissions>WriteDocument</Permissions>
```

<span data-ttu-id="cdc25-p102">Рекомендуется запрашивать разрешения по принципу _минимальной привилегии_. Т. е. следует запрашивать разрешения для доступа только к минимальному подмножеству API, необходимому надстройке для правильной работы. Например, если надстройке требуется только читать данные документа пользователя, оно не должно запрашивать большего, нежели разрешение **ReadDocument**.</span><span class="sxs-lookup"><span data-stu-id="cdc25-p102">As a best practice, you should request permissions based on the principle of  _least privilege_. That is, you should request permission to access only the minimum subset of the API that your add-in requires to function correctly. For example, if your add-in needs only to read data in a user's document for its features, you should request no more than the **ReadDocument** permission.</span></span>
