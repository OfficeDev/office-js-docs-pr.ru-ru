# <a name="office-add-ins-glossary"></a>Office глоссарий надстройок

Это глоссарий терминов, часто используемых во всей документации Office надстройки.

## <a name="add-in"></a>надстройка

Office надстройки — это веб-приложения, которые расширяют Office приложений. Эти веб-приложения добавляют новые функциональные возможности в Office, такие как привлечение внешних данных, автоматизация процессов или встраив интерактивные объекты в Office документах.

Office надстройки отличаются от надстройок VBA, COM и VSTO, так как они предлагают межплатформенную поддержку (как правило, веб-, Windows, Mac и iPad) и основаны на стандартных веб-технологиях (HTML, CSS и JavaScript). Основной язык программирования надстройки Office JavaScript или TypeScript.

## <a name="add-in-commands"></a>команды надстройки

**Команды надстройки** — это элементы пользовательского интерфейса, такие как кнопки и меню, которые расширяют Office пользовательского интерфейса для надстройки. Когда пользователи выбирают элемент команды надстройки, они инициируют действия, такие как запуск кода JavaScript или отображение надстройки в области задач. Команды надстройки дают возможность вашей надстройки выглядеть и чувствовать себя частью Office, что дает пользователям больше уверенности в вашей надстройки. [Дополнительные дополнительные Excel, PowerPoint и Word](../design/add-in-commands.md) и [надстройки](../outlook/add-in-commands-for-outlook.md) см. в Outlook надстройки.

См. также: [лента, кнопка ленты](#ribbon-ribbon-button).

## <a name="application"></a>приложение

**Приложение** относится к Office приложению. Приложения Office которые поддерживают Office надстройки, Excel, OneNote, Outlook, PowerPoint, Project и Word.

См. также: [клиент](#client), [хост](#host), [Office приложение, Office клиент](#office-application-office-client).

## <a name="application-specific-api"></a>API для приложений

API, определенные приложениям, предоставляют строго типные объекты, взаимодействующие с объектами, которые являются родными для определенного Office приложения. Например, вы вызываем Excel API JavaScript для доступа к таблицам, диапазонам, таблицам, диаграммам и более. В настоящее время API для приложений доступны для Excel, OneNote, PowerPoint, Visio и Word. [Дополнительные подробности см](../develop/application-specific-api-model.md). в примере модели API, определенной для приложений.

См. также: [Общий API](#common-api).

## <a name="client"></a>клиент

**Клиент** обычно ссылается на Office приложение. Приложения Office или клиенты, которые поддерживают Office надстройки, Excel, OneNote, Outlook, PowerPoint, Project и Word.

См. также: [приложение](#application), [хост](#host), [Office, Office клиент](#office-application-office-client).

## <a name="common-api"></a>Общий API

Общие API используются для доступа к таким функциям, как пользовательский интерфейс, диалоги и параметры клиентов, распространенные в нескольких Office приложениях. Эта модель API использует [обратные вызовы](https://developer.mozilla.org/docs/Glossary/Callback_function), позволяющие указывать только одну операцию в каждом запросе, отправляемом в приложение Office.

Общие API были введены Office 2013 г. и используются для взаимодействия с Office 2013 г. или более поздней. Некоторые распространенные API являются устаревшими API начала 2010-х. Excel, PowerPoint и Word имеют общие функции API, но большая часть этих функций была заменена или заменена моделью API, определенной для приложений. По возможности предпочтительны API, определенные приложениям.

Другие распространенные API, такие как общие API, связанные с Outlook, пользовательским интерфейсом и проверкой подлинности, являются современными и предпочтительными API для этих целей. Подробные сведения об объектной модели общего API см. в примере [объектной модели API Общего JavaScript](../develop/office-javascript-api-object-model.md).

См. также: [API для приложений](#application-specific-api).

## <a name="content-add-in"></a>надстройка контента

**Надстройки** контента — это веб-представления или представления веб-браузера, которые встроены непосредственно в Excel, OneNote или PowerPoint документы. Контентные надстройки предоставляют пользователям доступ к элементам управления интерфейсом, которые выполняют код для изменения документов или отображения данных. Используйте контентные надстройки, когда требуется внедрить функции непосредственно в документ. Дополнительные Office см. в Office [надстройки](../design/content-add-ins.md).

См. также: [веб-просмотр](#webview).

## <a name="content-delivery-network-cdn"></a>сеть доставки контента (CDN)

Сеть **доставки контента** **или CDN** это распределенная сеть серверов и центров обработки данных. Это обычно обеспечивает более высокую доступность ресурсов и производительность по сравнению с одним сервером или центром обработки данных.

## <a name="contoso"></a>Contoso

**Contoso** Ltd. (также известный как Университет Contoso и Contoso) — вымышленная компания, используемая Корпорацией Майкрософт в качестве примера компании и домена.

## <a name="custom-function"></a>настраиваемая функция

**Настраиваемая функция** — это функция, определяемая пользователем, которая упакована с Excel надстройки. Настраиваемые функции позволяют разработчикам добавлять новые функции, помимо типичных Excel, путем определения этих функций в JavaScript как части надстройки. Пользователи в Excel могут получать доступ к настраиваемой функции так же, как и к любой родной функции Excel. [Дополнительные дополнительные](../excel/custom-functions-overview.md) Excel см. в Excel Create custom functions.

## <a name="custom-functions-runtime"></a>настраиваемый запуск функций

**Настраиваемый запуск функций —** это время запуска JavaScript, которое выполняет только настраиваемые функции. Он не имеет пользовательского интерфейса и не может взаимодействовать с Office.js API. Если надстройка имеет только настраиваемые функции, это хорошее легкое время работы. Если пользовательские функции должны взаимодействовать с области задач или Office.js API, настройте общее время запуска JavaScript. Дополнительные сведения см. в статье [Настройка надстройки Office для использования общей среды выполнения JavaScript](../develop/configure-your-add-in-to-use-a-shared-runtime.md).

См. также: [время запуска JavaScript](#javascript-runtime), [совместное время работы JavaScript, общее время работы](#shared-javascript-runtime-shared-runtime).

## <a name="javascript-runtime"></a>Время запуска JavaScript

Время **запуска JavaScript —** это среда хост-браузера, в которой выполняется надстройка. В Office Windows и Office Mac время запуска JavaScript — это встроенный элемент управления браузером (или веб-просмотр), например Internet Explorer, Edge Legacy, Edge WebView2 или Safari. Различные части запуска надстройки в отдельных запусках JavaScript. Например, команды надстройки, настраиваемые функции и код области задач обычно используют отдельные время запуска JavaScript, если не настроить общее время запуска JavaScript. Дополнительные сведения см. в [Office браузерах](../concepts/browsers-used-by-office-web-add-ins.md), используемых Office надстройки.

См. также: [настраиваемый](#custom-functions-runtime) запуск функций, [совместное время запуска JavaScript, общее время работы](#shared-javascript-runtime-shared-runtime), [веб-просмотр](#webview).

Используйте области задач, когда не требуется или не может встраить функции непосредственно в документ. ## <a name="xll"></a>XLL

**Надстройка XLL** — это Excel надстройка, которая предоставляет определенные пользователем функции и имеет расширение **файла .xll**. XLL-файл — это тип динамического файла библиотеки ссылок (DLL), который может быть открыт только Excel. Файлы надстройки XLL должны быть написаны в C или C++. Настраиваемые функции — это современный эквивалент определенных пользователем функций XLL. Настраиваемые функции обеспечивают поддержку на различных платформах и совместимы с XLL-файлами. Дополнительные сведения см. в дополнительных сведениях о расширенных пользовательских функциях с помощью функций, определенных [пользователем XLL](/office/dev/add-ins/excel/make-custom-functions-compatible-with-xll-udf) .

См. также: [настраиваемая функция](#custom-function).
