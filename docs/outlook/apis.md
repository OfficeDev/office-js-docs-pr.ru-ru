## <a name="officejs-library"></a><span data-ttu-id="e57b0-105">Библиотека Office.js</span><span class="sxs-lookup"><span data-stu-id="e57b0-105">Office.js library</span></span>

> [!IMPORTANT]

## <a name="requirement-sets"></a><span data-ttu-id="e57b0-116">Наборы требований</span><span class="sxs-lookup"><span data-stu-id="e57b0-116">Requirement sets</span></span>

<span data-ttu-id="e57b0-p105">Чтобы указать, в каких клиентах Outlook появляется надстройка, задайте минимальную версию набора требований в манифесте. <span data-ttu-id="e57b0-p106">Указание набора требований не означает, что надстройке будут доступны API-интерфейсы только из этой версии. От набора требований зависит только то, в каких клиентах Outlook отображается надстройка.</span><span class="sxs-lookup"><span data-stu-id="e57b0-p106">Specifying a requirement set doesn't limit your add-in to the APIs in that version. The requirement set only controls which Outlook clients the add-in appears in.</span></span>

<span data-ttu-id="e57b0-125">Чтобы проверить доступность API-интерфейсов из набора требований, версия которого выше указанной в манифесте, вы можете использовать стандартный код JavaScript:</span><span class="sxs-lookup"><span data-stu-id="e57b0-125">To check the availability of any APIs from a requirement set greater than the one specified in the manifest, you can use standard JavaScript:</span></span>

> <span data-ttu-id="e57b0-126">Эти проверки необязательны для API, входящих в набор обязательных элементов версии, указанной в манифесте.</span><span class="sxs-lookup"><span data-stu-id="e57b0-126">These checks are not needed for any APIs that are in the requirement set version specified in the manifest.</span></span>

<span data-ttu-id="e57b0-127">Укажите минимальный набор обязательных элементов API для вашего сценария, без которого функции надстройки не будут работать.</span><span class="sxs-lookup"><span data-stu-id="e57b0-127">Specify the minimum requirement set that supports the critical set of APIs for your scenario, without which features of your add-in won't work.</span></span> <span data-ttu-id="e57b0-128">Набор обязательных элементов указывается в элементе `<Requirements>` манифеста.</span><span class="sxs-lookup"><span data-stu-id="e57b0-128">You specify the requirement set in the manifest in the `<Requirements>` element.</span></span> <span data-ttu-id="e57b0-129">Дополнительные сведения см. в статьях [Манифесты надстроек Outlook](manifests.md) и [Общие сведения о наборах обязательных элементов API Outlook](../reference/requirement-sets/outlook-api-requirement-sets.md).</span><span class="sxs-lookup"><span data-stu-id="e57b0-129">For more information, see [Outlook add-in manifests](manifests.md) and [Understanding Outlook API requirement sets](../reference/requirement-sets/outlook-api-requirement-sets.md).</span></span>
