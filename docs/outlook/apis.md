## <a name="officejs-library"></a><span data-ttu-id="9b6d7-106">Библиотека Office.js</span><span class="sxs-lookup"><span data-stu-id="9b6d7-106">Office.js library</span></span>

<span data-ttu-id="9b6d7-110">Добавьте ссылку на сеть CDN в теге `<script>`, вложенном в тег `<head>`, на веб-странице (HTML-, ASPX- или PHP-файле), где определяется пользовательский интерфейс надстройки.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-110">Reference the CDN in a `<script>` tag in the `<head>` tag of the web page (.html, .aspx, or .php file) that implements the UI of your add-in.</span></span>

<span data-ttu-id="9b6d7-p103">При добавлении новых API-интерфейсов не будет меняться URL-адрес Office.js. > [!IMPORTANT]
> <span data-ttu-id="9b6d7-113">Создавая надстройку для любого ведущего приложения Office, ссылайтесь на API JavaScript для Office из `<head>` раздела страницы.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-113">When developing an add-in for any Office host application, reference the Office JavaScript API from inside the `<head>` section of the page.</span></span> <span data-ttu-id="9b6d7-114">Это гарантирует, что API полностью инициализируется раньше всех элементов body.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-114">This ensures that the API is fully initialized prior to any body elements.</span></span> <span data-ttu-id="9b6d7-115">Ведущим приложениям Office необходимо, чтобы надстройки инициализировались в течение 5 секунд после активации.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-115">Office hosts require that add-ins initialize within 5 seconds of activation.</span></span> <span data-ttu-id="9b6d7-116">При превышении этого порога объявляется, что надстройка не отвечает, и отображается сообщение об ошибке.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-116">Crossing this threshold results in the add-in being declared unresponsive and an error message is displayed to the user.</span></span>

## <a name="requirement-sets"></a><span data-ttu-id="9b6d7-117">Наборы требований</span><span class="sxs-lookup"><span data-stu-id="9b6d7-117">Requirement sets</span></span>

<span data-ttu-id="9b6d7-p107">Указание набора требований не означает, что надстройке будут доступны API-интерфейсы только из этой версии. <span data-ttu-id="9b6d7-126">Чтобы проверить доступность API-интерфейсов из набора требований, версия которого выше указанной в манифесте, вы можете использовать стандартный код JavaScript:</span><span class="sxs-lookup"><span data-stu-id="9b6d7-126">To check the availability of any APIs from a requirement set greater than the one specified in the manifest, you can use standard JavaScript:</span></span>

> [!NOTE]

<span data-ttu-id="9b6d7-128">Укажите минимальный набор обязательных элементов API для вашего сценария, без которого функции надстройки не будут работать.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-128">Specify the minimum requirement set that supports the critical set of APIs for your scenario, without which features of your add-in won't work.</span></span> <span data-ttu-id="9b6d7-129">Набор обязательных элементов указывается в элементе `<Requirements>` манифеста.</span><span class="sxs-lookup"><span data-stu-id="9b6d7-129">You specify the requirement set in the manifest in the `<Requirements>` element.</span></span> <span data-ttu-id="9b6d7-130">Дополнительные сведения см. в статьях [Манифесты надстроек Outlook](manifests.md) и [Общие сведения о наборах обязательных элементов API Outlook](../reference/requirement-sets/outlook-api-requirement-sets.md).</span><span class="sxs-lookup"><span data-stu-id="9b6d7-130">For more information, see [Outlook add-in manifests](manifests.md) and [Understanding Outlook API requirement sets](../reference/requirement-sets/outlook-api-requirement-sets.md).</span></span>
